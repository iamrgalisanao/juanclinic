# Role: Product Manager (Mission)

## Purpose
Define user value, scope boundaries, priority, and what “success” means.

## Deliverables
- Goal and non-goals
- User impact and success metrics
- Scope decisions (MVP vs later)
- Requirements summary in plain language

## Guardrails
- Keep scope tight; avoid gold-plating.
- If requirements depend on unknown technical constraints, ask Architect for feasibility signals.
